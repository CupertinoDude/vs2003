  • Overview
  • Location
  • Building the sample
  • Running the sample

Overview

StringWalker shows how to handle Unicode surrogate pairs and combining characters when walking a string. It provides a class, named StringWalker that wraps the StringInfo.ParseCombiningCharacters method and handles the complexity of having to deal with surrogates and combining characters.

Location

This sample is located in the Technologies\Localization\StringWalker subdirectory of the .NET Framework SDK samples directory.

For Example:

C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\Localization\StringWalker

Building the Sample

In order to properly build and run this sample, the following steps need to be performed.

Type BUILD.bat from the command line.

For Example:

C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\Localization\StringWalker\VB>Build.bat

Running the Sample

In order to run this sample, follow the instructions listed below.

  • Type StringWalker.exe from the command line.

    For Example:

    C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\Localization\StringWalker\VB>StringWalker.exe

Notes

For the .Net Framework samples to function properly, your Path, Include, and Lib environment variables must be set correctly. In many cases, the SDK install program updates these environment variables for you. However, if you installed the .Net Framework SDK using Visual Studio .NET, or unselected "register environment variables" during setup, these environment variables will not be updated. You can set the Path, Include and Lib environment variables by running SDKVars.bat, located in the <SDKRoot>\Bin directory. SDKVars.bat must be executed in every command shell.
