Visual J# .NET Samples

Visual J# .NET samples consist of application and technology samples and are provided in a compressed file named VJSharpSamples.exe in the \Samples\VJSharp directory. To install these samples, you must first unzip this file by double clicking on it. To run each sample, proceed to the appropriate directory and consult the provided readme file. To run Visual J# samples, Visual J# .NET Redistributable Package 1.1 must be installed. For more information go to http://msdn.microsoft.com/vjsharp.

Application Samples

Samples that are modeled after complete applications that you can develop.

 

Sample Description

TypeFinder

Provides a command-line interface to get information about types from user's environment. This sample also provides general information about reflection.

WinTalk

Offers a brief introduction to Windows Forms and sockets, including other .NET Framework class library topics. The sample is a simple sockets chat application.

WordCount

Offers an introduction to the .NET Framework class library. Provides an application that counts words in a text file and organizes the results according to command-line arguments.

 

Technology Samples

The samples in this section illustrate the various technologies and features in the .NET Framework and Visual J#. They instruct on a single technology or a few related technologies. These samples are not strictly useful as complete applications and tend to exhibit limited error handling and object-oriented development.

 

Sample

Description

CLS Extender

Shows the authoring of .NET Framework properties, events, and delegates.

Cross language

Shows how Java-language syntax can be used in Visual J# to access classes written in other languages and how other languages can access Visual J# classes. Also shows how other languages can catch Java-language exceptions thrown by components written in the Java language.

PInvoke

Demonstrates how Visual J# can use the platform invoke services provided by the common language runtime to call unmanaged APIs.

RemotingSample

Shows how to use Visual J# to build a remoting service. This simple service manipulates a string passed to it as a parameter and returns the result back to the caller. It is important to note that the service must be hosted before a client can call it, which can be achieved by compiling the service as a DLL and registering the service in Internet Information Services (IIS)..

Attributes

Demonstrates how language applications compiled and run on Visual J# can use the .NET Framework attributes to provide enhanced functionality. Any attribute available in the .NET Framework can be attached to Visual J# programs. However, the Visual J# compiler does not support authoring (defining) attributes in Visual J# programs.

Enums

Demonstrates the use of .NET Framework enumerations. There is support to consume .NET Framework enumerations from Visual J#, but there is no support to author (define) enumerations.

EventsAndDelegates

Shows the use of .NET Framework delegates and events. You can consume .NET Framework delegates in Visual J#. The delegate or multicast keywords create delegates of type com.ms.lang.Delegate rather than of System.Delegate.

Minesweeper

Demonstrates how applications can target and consume the following .NET Framework features:

·         Events

·         Delegates

·         Value types

·         Properties

·         Attributes

Properties

Demonstrates the use of properties supported by the .NET Framework. There is support to get and set properties using the get_ and set_ property accessor methods.

ValueTypes

Demonstrates the use of value types supported by the .NET Framework. Visual J# does not support authoring (defining) value types. However, there is support to use the value types defined in the .NET Framework class library.
