Token2EE
Finds the memory address of common language runtime data for metadata token.
Syntax
!Token2EE <module name> <mdToken>
Parameters
<module name> Module name, including the .dll or .exe extension.
<mdToken> You can find the mdToken using DumpMD or DumpMT command.
Example 
0:000> !Token2EE debug.dll 06000001 
--------------------------------------
MethodDesc: 3e52e0
Name: [DEFAULT] [hasThis] String debug.MyClass.get_MyPropertyDll1()

0:000> !token2ee debug.dll 02000002  
--------------------------------------
MethodTable: 003e5344
EEClass: 031e3280
Name: debug.MyClass
Comments
Token2EE supports mdtMethodDef, mdtTypeDef and mdtTypeRef tokens.

For mdtMethodDef tokens it displays the method descriptor address and the method signature. Use the DumpMD command to get information about the method descriptor.

For mdtTypeDef and mdtTypeRef tokens, it displays the method table address, the EEClass address, and the class name.  Use the DumpMT command to get information about the method table and use the DumpClass command to get information about the EEClass.
