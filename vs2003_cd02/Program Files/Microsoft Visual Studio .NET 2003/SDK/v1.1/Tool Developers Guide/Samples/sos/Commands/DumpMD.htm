DumpMD
Dump Method Description information
Syntax
!DumpMD <addr>
Parameters

<addr>
Method Descr.

 Method descriptor address to dump information about
Example 
0:001> !DumpMD 0xda6a0b0 
Method Name : [DEFAULT] [hasThis] Void System.Web.UI.WebControls.Button.OnClick(Class System.EventArgs)
MethodTable da6a130
Module: 1433e0
mdToken: 060012d6 (c:\windows\assembly\gac\system.web\1.0.3300.0__b03f5f7f11d50a3a\system.web.dll)
Flags : 80
Method VA : 0de8cdf8
Output
Method Name

Method name including parameters. [hasThis] indicates that the method requires an instance of the class to be called. The calling convention can be one of the following values

[DEFAULT]
[C]
[STDCALL]
[THISCALL]
[FASTCALL]
[VARARG]
[FIELD]
[LOCALSIG]
[PROPERTY]
[UNMANAGED]
MethodTable Method table the method descriptor is in.  Use the DumpMT command to get more information about the method table.
Module Module the method descriptor is in.  Use the DumpModule command to get more information about the module.
mdToken Metadata token for the method descriptor.
Flags Flags associated with the method descriptor.  Bit-masked DWORD.
Method VA
or
IL RVA 		If Method VA (Virtual Address) is displayed this is the location in native code that the method has been JIT-compiled to.  You can use the u command on this address to display the assembly.  If IL RVA (Intermediate Language Relative Virtual Address) is displayed, the method has not been JIT-compiled yet.

Comments

A method descriptor  address can be obtained using the DumpStack, DumpMT, Name2EE or IP2MD commands.
