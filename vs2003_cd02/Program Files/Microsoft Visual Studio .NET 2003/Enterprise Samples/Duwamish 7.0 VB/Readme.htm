Known Issues

Setup fails when entering database name

Setup fails after entering a database name that contains spaces or other non-alphanumeric characters.

·         Do not use database names with spaces or quotes. Use only alphanumeric characters.

Running Duwamish 7.0 from Visual Studio .NET

To run Duwamish 7.0 from the Visual Studio .NET integrated development environment (IDE), you need to:

1.      Make the Web project the startup project. In Solution Explorer, right-click the Web project and select Set as Startup Project.

2.      Make Default.aspx the start page. Right-click the file Default.aspx and select Set as Start Page.

Security - user ID and password with the Duwamish 7.0 CatalogService Web service

When invoking the GetBooksByTopicSecure method of the Duwamish 7.0 CatalogService, user name and password are sent as clear text in the HTTP Get request:

 

http://localhost/Duwamish7/service/catalogservice.asmx/GetBooksByTopicSecure?userName=someone@example.com&password=PasswordHere&topicName=a&maxQty=23

 

Resolve this security problem by adding a <webServices> section to the Web.config file:

<configuration>
…

<system.web>
…

        <webServices>

            <protocols>

              <remove name="HttpGet"/>

            </protocols>

        </webServices>

 

As a result, requests are sent as HTTP Post instead of HTTP Get and the user name and password will not be visible.

Adjusting currency amounts to locale appropriate amounts

The database of the Duwamish 7.0 application is only localized for the following four languages:

·        Japanese

·        Traditional Chinese

·        Simplified Chinese

·        Korean

Because the application retrieves currency symbols from the operating system, pricing information displayed on the Web pages is unrealistic in some countries. For example, if a particular book costs 4.95 U.S. dollars, when the application runs on a computer in Italy, the price is shown as 4.95 Italian Lira. You can adjust the pricing information by running the following query against the Duwamish7 or Duwamish7VB database.

 

UPDATE    Items

SET              UnitCost = UnitCost *1144.17, UnitPrice = UnitPrice * 1144.17

 

Where 1144.17 is the foreign currency (Italian Lira in this example) equivalent of 1 U.S. dollar.

 

To run the query, open SQL Server Query Analyzer, select the appropriate database, paste the query with the right conversion rate into the query window and press the execute button on the toolbar.

Changes made to the Duwamish policy file (Duwamish.tdl) lost on reinstall

The Duwamish Enterprise Templates policy file is installed in the default Enterprise Templates directory structure: [Location where Visual Studio is installed]\EnterpriseFrameworks\Policy. Changes made to the Duwamish.tdl file will be lost if the application is reinstalled, as the installer replaces the file without warning. If you must reinstall Duwamish 7.0 and want to keep your changes to Duwamish.tdl, save your modified version of Duwamish.tdl to another directory before reinstalling. After reinstalling Duwamish 7.0, replace the reinstalled copy of Duwamish.tdl with your modified version.

Cannot find page – link incorrect

The link for a specific page shows at the localhost as the server name in the URL. When clicking on it, the resulting request yields “can’t find page.” If, after application installation, the site is initially accessed using localhost instead of a fully-qualified URL, later access to the site from a remote computer results in incorrect links that contain localhost in the URL.  This issue is due to page caching. 

To remedy this problem, either reset Internet Information Services (IIS) by typing iisreset from the command line or wait for the cache to timeout.

Installation Prerequisites

Duwamish 7.0 has the following installation prerequisites:

·         Operating System: Microsoft Windows 2000 (all versions) or Microsoft Windows XP Professional

·         Microsoft SQL Server 2000 or SQL Server 7.x, set to mixed-mode authentication with Service Pack 2 applied

·         Microsoft Internet Explorer 5.5 or greater

·         Microsoft Data Access Controls (MDAC) 2.7 or greater

·         Microsoft .NET Framework SDK

·         Microsoft Internet Information Services (IIS) 5.0 or greater

In addition, if you want to build the application using the installed project files, you also need to install:

·         Visual Studio .NET Enterprise Architect or Visual Studio .NET Enterprise Developer

Command Line Build

All of the following instructions assume that you are in the installation directory for Duwamish 7.0.

To compile from the command line, navigate to the directory in which Duwamish 7.0 is installed and type:

nmake /a -f duwamish.mak CFG=<config> all

Where <config> is one of Debug or Release. If you omit the CFG=, then Debug is the default. Note that both Debug and Release binaries are built into the same output directory - [Location where Duwamish 7.0 is installed]\Web\Bin – therefore it is strongly recommended that you use the /a (force make) option as shown above.

Removing and Repairing Duwamish 7.0

To repair Duwamish 7.0 

1.      Locate the Duwamish folder. The default locations are: 
[Directory where Visual Studio is installed]\Enterprise Samples\Duwamish 7.0 CS 
[Directory where Visual Studio is installed]\Enterprise Samples\Duwamish 7.0 VB 

2.      Select and run the Duwamish Windows Installer Package file, Duwamish.msi.
   The Welcome to the Duwamish Setup Wizard page appears.

3.      If Repair is not selected by default, select it. This utility restores Duwamish 7.0 to its original state if application components become corrupted.

4.      Select Finish to repair the application.

To remove Duwamish 7.0

The removal utility does not remove the Duwamish database.

To completely remove the application, perform the following steps.

1.      Locate the Duwamish folder. The default locations are:
[Directory where Visual Studio is installed]\Enterprise Samples\Duwamish 7.0 CS 
[Directory where Visual Studio is installed]\Enterprise Samples\Duwamish 7.0 VB

2.      Select and run the Duwamish Windows Installer Package file, Duwamish.msi.
   The Welcome to the Duwamish Setup Wizard page appears.

3.      Select Remove and Finish to remove components.

4.      Open SQL Server Enterprise Manager and select the database (Duwamish7 or Duwamish7vb) containing the Duwamish 7.0 catalog tables.

5.      Right-click on the appropriate database and select Delete.

6.      Select OK to delete the database.

7.      Select Security Logins.

8.      Select Duwamish7_login and delete it.
