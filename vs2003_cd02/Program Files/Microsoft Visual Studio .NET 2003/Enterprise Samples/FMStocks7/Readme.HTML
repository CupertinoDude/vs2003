Known Issues

Setup fails when entering database name

Setup fails after entering a database name that contains spaces or other non-alphanumeric characters.

·         Do not use database names with spaces or quotes. Use only alphanumeric characters.

IIS Server fails to respond to page requests after setup fails

If Setup fails for any reason and there was no subsequent successful re-install of Fitch and Mather 7.0, the Internet Information Services (IIS) Server will not be restarted.

1.      To restart IIS, click Start and then Run.

2.      In the Run dialog box, type iisreset.

Running Fitch and Mather 7.0 from Visual Studio .NET

To run Fitch and Mather 7.0 from the Visual Studio .NET integrated development environment (IDE), you need to:

1.      Make the Web project the startup project. In Solution Explorer, right-click the Web project and select Set as Startup Project. 

2.      Make Login.aspx the start page. Right-click the file Login.aspx and select Set as Start Page. 

Intermittent General Network Error when selling more shares than the user owns

If you try to sell more shares than the logged on user owns, you might experience intermittent general networks errors. This is a known bug in SQL Server 2000 and a fix is available in a service pack. Refer to the Installation Prerequisites section for more information on the latest service packs to install.

Cannot step into the FMStocks7.DAL.dll

There is a limitation in Visual Studio .NET that does not allow the user to step into ServicedComponents where no breakpoint has been previously set.  Since all of Fitch and Mather’s DAL components are ServicedComponents, it is not possible to step into this code unless a breakpoint is present.

Cannot load symbols for FMStocks7.DAL.dll

When trying to debug the DAL (by setting a breakpoint inside the DAL code, see above) the symbols cannot be found.  This occurs because the symbol file FMStocks7.DAL.pdb is located in the Debug subdirectory of the DAL project; the system is looking for the .pdb file in the global assembly cache because the assembly is loaded from the global assembly cache.

To resolve this problem, remove the FMStocks7.dal assembly from the global assembly cache by running the following commands from the command line:

iisreset
Gacutil –u FMStocks7.DAL

It is a best practice to add ServicedComponents to the global assembly cache in case they are run as a Server COM+ Application.  However, for Library COM+ applications that run in-process, it is not required.  For more information, see Registering Serviced Components in the Visual Studio .NET documentation. 

Deploying to an Application Center Cluster controller fails

Deployment to an Application Center Cluster controller will fail to create the Application Center application if the administrator is not logged on to the target machine (the application tier cluster controller).

1.      Log the admin user onto the target machine.

2.      Re-run the FMS deployment executable, FMS.exe. For more information, see Scenario: Web External Hosting/Perimeter Network in the Visual Studio .NET documentation.

3.      Log off the admin user from the target machine.

Running the Legacy Interop scenario

To run the application as described in the Visual Studio .NET documentation in Scenario: Legacy Interop, you must change the security configuration.  By default all ASP.NET applications run under the privileges of the ASPNET account.  To enable features of this application, change the security account as follows.

·         Change <processModel userName="MACHINE"> to <processModel userName="SYSTEM"> in <Windows directory>\Microsoft.NET\Framework\<version>\CONFIG\machine.config on all machines participating in the distributed scenario.

Caution   Changing the processModel from MACHINE to SYSTEM causes all ASP.NET applications to run under the SYSTEM account.  This is a higher-privileged account and should be used with caution.

Legacy Interop scenario with Application Center only load balances to a single machine

In this configuration, the GAM is never called on more than one machine in the Application Center Cluster. For more information see Distributed Deployment Scenarios in the Visual Studio .NET documentation. To correct this problem, perform the following steps on the IIS machine after deployment:

1.      Open the Component Services MMC.

2.      Navigate to Computers/My Computer/COM+ Applications.

3.      Navigate to the FMStocks7.GAM application, right-click and select Shut Down.

4.      Open the FMStocks7.GAM application properties.

5.      On the Activation tab, change the Remote Server from <App cluster controller machine name> to <App cluster virtual IP address>.

Error Viewing resource files (.resx) in Data viewer of Resource editor

When opening one of the following resource files, an error is displayed when the Data tab in the resource editor is selected.  The XML view is not affected.

FMStocks7.BLL.resx

FMStocks7.DAL.resx

FMS.resx

FMStocks7.Web.resx

To resolve this problem:

1.      Double-click the file to open it in the Resource editor.

2.      Select XML view. 

3.      Remove the targetNameSpace entry in the first line.
Before change:
      <xsd:schema id="root" targetNamespace="" xmlns="" xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:msdata="urn:schemas-microsoft-com:xml-msdata">
After change:
      <xsd:schema id="root" xmlns="" xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:msdata="urn:schemas-microsoft-com:xml-msdata">

4.      Save file and reopen.

Error in Buy Stock, Sell Stock, and Create New Account on Windows Server 2003 family

Fitch and Mather 7.0 uses distributed transactions between two different databases. This requires the DTC to have Network Access and Network Transactions enabled, which are disabled by default on Windows Server 2003.

To correct the problem, follow these steps:

1.      Open the Component Services MMC (under Administrative tools).

2.      Open Computers.

3.      Open the Properties page for My Computer (by right-clicking My Computer).

4.      Go to the MSDTC tab and click Security Configuration.

5.      In the Security Configuration dialog box, select Network DTC Access and Network Transactions.

6.      Restart IIS by running: iisreset.exe from the command line.

Installation Prerequisites

Fitch and Mather 7.0 has the following installation prerequisites:

·         Operating System: Microsoft Windows 2000 (all versions) or Microsoft Windows XP Professional

·         Microsoft SQL Server 2000 or SQL Server 7.x, set to mixed-mode authentication with Service Pack 2 applied

·         Microsoft Internet Explorer 5.5 or greater

·         Microsoft Data Access Controls (MDAC) 2.7 or greater

·         Microsoft .NET Framework SDK

·         Microsoft Internet Information Services (IIS) 5.0 or greater

In addition, if you want to build the application using the installed project files, you also need to install:

·         Visual Studio .NET Enterprise Architect or Visual Studio .NET Enterprise Developer

Removing and Repairing Fitch and Mather 7.0

To repair Fitch and Mather 7.0 

1.      Locate the FMStocks7 folder. The default location is [Directory where Visual Studio is installed]\Enterprise Samples\FMStocks7. 

2.      Select and run the FMStocks7 Windows Installer Package file, FMStocks7.msi.
   The Welcome to the Fitch and Mather Setup Wizard page appears.

3.      If Repair is not selected by default, select it. This utility restores Fitch and Mather 7.0 to its original state if application components become corrupted.

4.      Select Finish to repair the application.

To remove Fitch and Mather 7.0

1.      Locate the FMStocks7 folder. The default location is [Directory where Visual Studio is installed]\Enterprise Samples\FMStocks7. 

2.      Select and run the FMStocks7 Windows Installer Package file, FMStocks7.msi. 
The Welcome to the Fitch and Mather 7.0 Setup Wizard page appears. 

3.      Select the Remove option. 

4.      Select Remove again to remove components. 

Note   The Setup Wizard will remove the FMStocks7 and FMStocks7_GAM SQL Server or Oracle databases as part of the uninstall process.

Oracle8 Setup Instructions (configuration only supported on Oracle8 versions)

To setup the Oracle8 Client on Windows 2000

1.      Install Oracle8 Client Version 8.0.5.0.0 for Windows NT.

2.      Install the Oracle8 8.0.5.2.6 client patch for Windows NT. 

a.      Download the patch file from ftp://oracle-ftp.oracle.com/server/patchsets/wgt_tech/server/windowsNT/80patchsets/80526 to your Oracle root directory (e.g., C:\OraNT). 

b.      Stop the OracleClientCache80 service and any other Oracle8 services that might be running. 

c.      Run the patch file to extract the setup files to folder 80526. 

d.      Run the 80526\Setup.exe program. Choose everything in the list of components the Oracle Installer presents. The Oracle Installer will only install those components present on your machine. 

e.      Start the OracleClientCache80 service and any other Oracle8 services that you might have stopped. 

3.      Open the Registry key below using Regedit: 
 [HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\MSDTC\MTxOCI] 
Double-click on MTxOCI. In the right pane, double-click on each of the Values Names below and enter the corresponding Value Data. 
ValueName     ValueData 
OracleXaLib     Xa80.dll 
OracleSqlLib     Sqllib80.dll 
OracleOciLib     Oci.dll 

4.      Place this entry in TNSNames.ora found in C:\OraNT\Net80\Admin directory (replace C:\OraNT with your Oracle installation directory): 
FMSTOCKS7_GAM.WORLD = (DESCRIPTION = (ADDRESS = (PROTOCOL = TCP)(HOST = <SERVERNAME>)
(PORT = 1521)) (CONNECT_DATA = (SID = ORCL))) 

To set up Oracle8 Server on Windows 2000

1.      Install the Oracle8 8.0.5.2.6 patch for Windows NT. 

a.      Download the patch file from ftp://oracle-ftp.oracle.com/server/patchsets/wgt_tech/server/windowsNT/80patchsets/80526 to your Oracle root directory (e.g., C:\orant). 

b.      Stop any other Oracle8 services that might be running. 

c.      Run the patch file to extract the setup files to folder 80526. 

d.      Run the 80526\Setup.exe program. 

e.      Start any other Oracle8 services that you might have stopped. 

2.      Open SQL Plus 8.0, login as SYS/CHANGE_ON_INSTALL (or your SYS user password)/ FMSTOCKS7_GAM.WORLD. 

3.      Make sure you have DBA_PENDING_TRANSACTIONS views created on your Oracle 8/8i database. Type “SELECT * FROM USER_VIEWS WHERE VIEW_NAME = 'DBA_PENDING_TRANSACTIONS';” 

4.      The Oracle system administrator must grant SELECT access to the public for the DBA_PENDING_TRANSACTIONS view. To grant the privileges type “GRANT SELECT ON DBA_PENDING_TRANSACTIONS TO PUBLIC;” 

5.      You should now be able to connect to the Oracle database under the transaction mode. For more details about configuring the Oracle server and Oracle client for transaction support, see the following document.
