Automatic Log On Readme

Issues you should be aware of when using Automatic Log On are listed below.

1. Known Issues

1.1. Password left behind when system fails during setup

If the machine shuts down unexpectedly during setup and you have chosen to use the Automatic Log On feature, your password information may remain in temporary setup files as well as the system registry.

To remove temporary files
  1. On the Start menu, choose Run.
  2. In the Run dialog box, type: %TEMP%.
  3. Select all files that begin with the letters "SIT" and delete them.

On Windows 2000 automatic log on information may be left in the registry, depending on when the system failed.

To remove automatic log on from the registry on Windows 2000
  1. On the Start menu, choose Run.
  2. In the Run dialog box, type: regedit.
  3. Navigate to the HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon key.
  4. Set the AutoAdminLogon value to 0.
  5. Delete DefaultPassword.

1.2. Lock Computer is not set on restart

When you use automatic log on, setup does not lock your machine after setup completes, even if the machine had been locked previously.

1.3. Blank passwords are not supported

Your password must have at least one character for automatic log on to function.

1.4. Log on banners and legal notices interrupt automatic log on

Automatic log on cannot ignore legal notices or log on banners that display when Windows launches. You must manually dismiss these messages for setup to continue. If your system administrator has specified that legal notices or log on banners appear, it is recommended that you do not use the automatic log on feature.

1.5. Machines running third-party client networking software are not supported

Setup does not automatically log on after a restart on machines running third-party client networking software. It is recommended that you do not use this feature if you have this type of software installed.
