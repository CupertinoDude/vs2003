Microsoft Visual Studio .NET Prerequisites

Components with Languages Version Conflicts

Visual Studio .NET setup requires that certain prerequisites be installed in the same language version as the language version of your operating system. You must install the matching language versions for these components before setup can continue.

Other prerequisites can automatically be installed in the default language version, English, or manually installed in language version that matches the language version of your operating system.

Why do I have to manually install matching language versions?

Visual Studio .NET setup does not contain all of the possible language versions for prerequisites. The matching language versions for these components are available from the Internet. Setup cannot automatically download these components for you.
How do I install matching language versions?

Return to the Components with Language Version Conflicts dialog box and click Download Components. A Web page will appear, listing links to download sites on the Internet. After you have installed the required matching language version, you can return to setup by running Setup on Disk 1 of the Visual Studio product you are installing.
What happens if I choose not to install optional matching language versions?

Setup will install the default language version of the component, which means that the text in the user interface for that component will be in English.
