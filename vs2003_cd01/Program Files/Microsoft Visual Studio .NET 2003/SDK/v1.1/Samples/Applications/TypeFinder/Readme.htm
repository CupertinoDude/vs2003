Overview

This sample utility provides the user with a command line interface to provide information about types in their environment. Sometimes locating a specific type can be difficult (what namespace is it in, which DLL?) and once you have located the type - the documentation may be unavailable or it is out of date. This utility provides a simple way for a developer to determine what types are available, what module they are in and what interfaces, methods, fields, properties and events are available on the type. This sample also provides a peek into reflection.

Topics

The following bullets briefly describe the classes and technologies used by this sample.

  • Reflection -
    • Assembly - Used to load assemblies into the AppDomain so that they can be searched for types.
    • Module - Used to get types from the Assembly/Module for comparison to the search string.
    • Type - Used to get type information such as its name, namespace, and members.
    • PropertyInfo - To find information about properties in types.
    • EventInfo - Used to find information about events in types.
    • FieldInfo - Finds information about fields in types.
    • MethodInfo - Used to finds information about methods in types.
  • IO -
    • TextWriter - Used by the IndentedWriter sample type to output (to the console, by default) in a generic fashion.
  • Text -
    • StringBuilder - Used by the IndentedWriter sample type create a string.
    • String - Used throughout the sample to find format strings, find sub-strings, upper-case strings etc.
  • Collections -
    • ArrayList - Used by the sample to manage a growable list of strings.
  • Registry -
    • Registry - Used to create an instance of the RegistryKey type for a subkey of the LocalMachine key.
    • RegistryKey - Used to read values from a key in the registry.

Location

This sample is located in the Applications\TypeFinder subdirectory of the .NET Framework SDK samples directory.

For Example:

C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Applications\TypeFinder

Building the Sample

In order to properly build and run this sample, the following steps need to be performed.

Type BUILD.bat from the command line.

For Example:

C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Applications\TypeFinder\VB>Build.bat

Running the Sample

In order to run this sample, follow the instructions listed below.

  • Type FindType.exe from the command line for a complete list of command line options for the utility.

    For Example:

    C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Applications\TypeFinder\VB>FindType.exe

  • To find the location of each type with a word in its name type FindType.exe [word] from the command line.

    For Example:

    C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Applications\TypeFinder\VB>FindType.exe String

Notes

For the .Net Framework samples to function properly, your Path, Include, and Lib environment variables must be set correctly.
