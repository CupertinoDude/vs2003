Overview

This sample demonstrates the use of structured exception handling in various programming languages. In addition to throwing and catching exceptions, this sample also shows how to install an unhandled exception handler which is called when an exception goes "un-caught".

Topics

The following bullets briefly describe the classes and technologies used by this sample.

  • Exceptions -
    • ApplicationException - The sample uses the ApplicationException type as the base type for its custom exception. Custom exceptions in your applications should be derived from ApplicationException.
    • Exception - The base of all exception classes, this class is used by the sample to catch any exception and display information about the exception.
  • Threading -
    • Thread - This class is used to get the AppDomain for the sample.
  • Reflection -
    • AppDomain - This class is used by the sample to set the unhandled exception handler event.
  • Events and Delegates -
    • UnhandledExceptionEventHandler - The delegate type for setting a function to be called when an exception goes unhandled.
  • Windows Forms -
    • MessageBox - The sample uses this type to display a simple message box in the unhandled exception handler.

Location

This sample is located in the Technologies\Exceptions subdirectory of the .NET Framework SDK samples directory.

For Example:

C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\Exceptions

Building the Sample

In order to properly build and run this sample, the following steps need to be performed.

Type BUILD.bat from the command line.

For Example:

C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\Exceptions\VB>Build.bat

Running the Sample

In order to run this sample, follow the instructions listed below.

  • Type Except.exe from the command line.

    For Example:

    C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\Exceptions\VB>Except.exe

Notes

For the .Net Framework samples to function properly, your Path, Include, and Lib environment variables must be set correctly. In many cases, the SDK install program updates these environment variables for you. However, if you installed the .Net Framework SDK using Visual Studio .NET, or unselected "register environment variables" during setup, these environment variables will not be updated. You can set the Path, Include and Lib environment variables by running SDKVars.bat, located in the <SDKRoot>\Bin directory. SDKVars.bat must be executed in every command shell.
