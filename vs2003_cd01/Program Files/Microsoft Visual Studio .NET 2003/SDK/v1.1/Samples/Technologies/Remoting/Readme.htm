Remoting Roadmap

The .Net Framework SDK provides various ways to learn about Remoting.

Introductory Remoting Concepts
If you're new to Remoting and would like a walkthrough of basic Remoting concepts, the QuickStart tutorials are a good place to start. 

  • The QuickStart tutorials can be viewed on the internet from the Online QuickStart website.

  • Click on the Start the Common Tasks QuickStart Tutorial link on that page.

  • From the index on the left, click on the Remoting link. 

  • The QuickStart tutorials are designed to be viewed in a sequential order.  Each subsequent topic builds upon the previous lesson.  Start at the Remoting Overview and work your way through the end of the Remoting Section.

Remoting in the .Net Framework
If you're familiar with Remoting concepts and would like a description of how Remoting works in the .Net Framework, see the Accessing Objects in Other Application Domains Using .NET Remoting section of the help documentation.  This section provides in-depth details of how the fundamental Remoting concepts are implemented in the .Net Framework. 

The following SDK samples correspond to the concepts covered in this section:

  • Remoting Hello

  • Remoting Events

  • Remoting Collections

  • Remoting Objects

Advanced Remoting Concepts
For information on advanced Remoting concepts, review the information in the .Net Framework Developer's Specifications section of the help documentation.  Here, concepts such as Remoting Metadata, Custom Channels and Asynchronous Programming are covered in detail.

The following SDK samples correspond to the concepts covered in this section:

  • Async Delegate

  • Call Context

  • Channel Sinks

  • Custom Proxies

  • Remoting COM

  • Serialization

Security in the Remoting Samples
The remoting samples do not illustrate how to do secure remoting. Users should be aware that virtual directories added to IIS to run the remoting samples can be executed by others if the machine is connected to a network. For information on how to apply security to IIS virtual directories, please review the security section of the IIS Help Documentation. A paper demonstrating secure .NET Framework remoting will be published online.
