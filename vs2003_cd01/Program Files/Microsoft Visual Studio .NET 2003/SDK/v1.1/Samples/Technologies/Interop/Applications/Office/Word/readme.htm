Microsoft Word Interoperability Sample

This document contains the release information for the Word Interoperability sample.  You will find the following sections below:

 

  • Sample Overview
  • Requirements
  • Location of the Sample
  • Building the Sample
  • Running the Sample

 

Sample Overview

This sample demonstrates using Microsoft Word from a managed application written in C#. Microsoft Word is the document component of Microsoft Office. Microsoft Word’s object model is exposed through its Automation type library msword.olb. The purpose of this sample is to demonstrate that a managed application can interact with Microsoft Word as a COM server. 

Requirements

This sample requires Microsoft Office XP with SP1 or later to be installed on your system. 

Location of the Sample

This sample is located in the Interop\Applications\Office\Word directory of the .NET Framework SDK samples directory.

For Example:

C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\Interop\Applications\Office\Word

 

Building the Sample

In order to properly build and run this sample, the environment variable, MSWordTlb, must be set up to point to the location of Word’s type library:

 

  • Set the variable to the location of msword.olb.

For Example:

set MSWordTlb=C:\Program Files\Microsoft Office\Office10\msword.olb

 

·         Once the environment is properly established then type NMAKE from the command line.

For Example:

NMAKE

 

Running the Sample

In order to run this sample, follow the instructions listed below.

  • Type WordApp from the command line.

For Example:

WordApp

 

·         At the end of execution the sample closes the Internet Explorer window opened to show hyperlink properties and methods of a document.
