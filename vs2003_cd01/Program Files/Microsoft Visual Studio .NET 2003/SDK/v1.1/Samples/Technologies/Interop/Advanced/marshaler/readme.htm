CLR Marshaler Sample

This document contains the release information for the CLR Marshaler sample.  You will find the following sections below:

  • Sample Overview
  • Requirements
  • Location of the Sample
  • Build the Sample
  • Running the sample

 

Sample Overview

The Marshaler sample provides an alternative marshaling mechanism to the ole automation mechanism that the CLR uses.  It is 100% compatible with the ole automation scheme, in as much as any interface marshalable by ole automation will be marshalable by this marshaler.  This marshaler provides enhanced compatibility with the CLR's assembly versioning scheme.

The ole automation marshaler relies on registered typelibs to operate correctly.  This marshaler has the CLR locate a typelib for use with the assembly.  If none can be found then the CLR will generate one that matches on the fly.

The comreg sample with the /p option can be used to register components for use with this marshaler.

The tlbgen sample can be used to embed a type library within an assembly.

 

Requirements

This utility will only operate correctly on any version of Windows with the Common Language Runtime installed.  You will also need a C++ compiler or the Microsoft platform SDK for the Windows header files.

 

Location of the Sample

This sample is located in the …\Interop\Advanced\Marshaler directory of the .NET Framework SDK samples directory.

For Example:

C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\Interop\Advanced\Marshaler

  

Building the Sample

In order to properly build and run this sample:

  1. At the command line enter the following:
    • NMAKE All

 

Running the Sample

The sample is a dll and should be registered using the winnt utility RegSvr32.exe

To register at the command prompt enter:

  • regsvr32 marshaler.dll

To unregister at the command prompt enter:

  • regsvr32 /u marshaler.dll
