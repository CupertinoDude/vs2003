Sample Overview

This sample demonstrates the use of an unmanaged COM component from managed code and the use of a managed component from COM.    

Requirements

Portions of the sample require .NET Framework, Visual C++ 6.0, Visual Basic 6.0, and Internet Information Services (IIS).

Note:   The compiled DLL and EXE files are available for unmanaged portions of the sample in case you dont have Visual Studio 6.0.

Location of the Sample         

This sample is located in the Interop\Applications\LoanApps directory of the .NET Framework SDK samples directory.

For Example:

C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\Interop\Applications\LoanApps

This sample contains the following directory structure:

COMtoNET  COM clients and .NET server
     \LoanLib  A managed implementation of the Loan class written in C#
     \WinLoan  A VB6 application that uses the managed Loan class to do a loan amortization
     \ConLoan - A Console application written in C++ that that uses the managed Loan class to do a loan amortization

NETtoCOM  .NET clients and COM server
     \LoanLib - An unmanaged implementation of the Loan class written in C++ using ATL
     \WinLoan - A Windows Forms application written in C# that that uses the unmanaged Loan class to do a loan amortization
     \ConLoan - A Console application written in C# that that uses the unmanaged Loan class to do a loan amortization
     \LoanWebService  A WebService that exposes the unmanaged Loan class to clients
     \WebLoan  An ASP.NET application written in C# that uses the unmanaged Loan class to do a loan amortization    

Building and Running the Sample

In order to run this sample, follow the instructions listed below.  In each of the following demos Balance, Rate, Term and Payment are defined as:

     Balance = Loan Opening balance (i.e. 10000.00)
Rate = Monthly interest rate (i.e. 1.0)
Term = Term in months (i.e. 60)
Payment = Monthly payment (i.e. 222.44)

Three of the above values must be supplied.   The fourth value will be computed.   The risk rating text field is added to show how the strings are marshaled. It has no influence on the result.

Note:   Since this is intended to be only a sample, no error checking is done.   Invalid input will crash the sample.

Step 1 - Using the COM loan class from managed code

·         Open the LoanLib.dsw file in the NETtoCOM\LoanLib directory with Visual Studio 6.   Choose Build\Rebuild All from the menu to build and register the COM server.  

·         Go to the NETtoCOM\ConLoan directory and type nmake all at the prompt to build the .NET console client. 

·         Run ConLoan in the NETtoCOM\ConLoan directory with the following command line:

ConLoan   Balance Rate Term Payment

Three of the above values must be supplied.   The fourth value must be zero.

·         Go to the NETtoCOM\WinLoan directory and type nmake all at the prompt to build the .NET Windows Forms client. 

·         Run WinLoan.exe in the NETtoCOM\WinLoan directory.

First supply any three of the values and click Compute to calculate the fourth.  Then click Amortize to generate the amortization schedule.

·         Go to the NETtoCOM\LoanWebService directory and type nmake all at the prompt to make the assembly needed for the WebService and to create the new virtual directory. 

·         Click on this link http://localhost/NetSDK/LoanWebService/LoanLib.asmx to see the Loan WebService.

·         Click on Amortize or Calculate link, supply all four parameters (one must be zero), and click Invoke. The XML file with a result will be generated.

·         Go to the NETtoCOM\WebLoan directory and type nmake all at the prompt to make the assembly needed for the ASP.NET application and to create the new virtual directory. 

·         Click on this link http://localhost/NetSDK/WebLoan/WebForm1.aspx to see the ASP.NET page.

First supply Open Balance, Rate, and Term values and then click Calculate to calculate the fourth value and the amortization schedule.

 

Step 2 - Using the .NET loan class from unmanaged code

·         Go to the COMtoNET\LoanLib directory and type nmake all at the prompt to build the .NET server.   It will also add the assembly to the global assembly cache, produce a COM type library, and register it with the system.

·         Open the ConLoan.dsw file in the COMtoNET\ConLoan directory with Visual Studio 6.   Choose Build\Rebuild All from the menu to build the C++ client.  

·         Run ConLoan in the COMtoNET\ConLoan directory with the following command line:

ConLoan   Balance Rate Term Payment

Three of the above values must be supplied.   The fourth value must be zero.

·         Open the VBLoan.vbp file in the COMtoNET\WinLoan directory with Visual Basic 6.0.  

·         The reference to the LoanLib type library, produced in the previous step, is already added to the project. You can check this by selecting Project\References from the menu.   

·         Press F5 to run the project. Alternatively, select File\Make LoanApp.exe to create an executable and run this executable.   

First supply any three of the following values and click Compute to calculate the fourth.   Then click Amortize to generate the amortization schedule.
