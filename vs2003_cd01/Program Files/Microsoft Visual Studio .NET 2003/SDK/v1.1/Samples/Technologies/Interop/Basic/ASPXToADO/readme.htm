ADO Interoperability Sample

This document contains the release information for the ADO Interoperability sample.  You will find the following sections below:

 

  • Sample Overview
  • Requirements
  • Location of the sample
  • Building the sample
  • Running the sample

 

 

Sample Overview

This sample demonstrates the use of an existing unmanaged COM component (ADO) from ASP.NET page using C# and VB.NET. It interacts with the ADO COM server and uses its objects and methods.

 

Requirements

This sample requires Windows NT4 or greater, IIS Services and SQL Server 7.0 to be configured and running on your system or to have a remote access to SQL Server 7.0, and Microsoft ActiveX Data Objects version 2.5 or later. 

 

Location of the Sample

This sample is located in the Interop\Basic\ASPXToADO directory of the .NET Framework SDK samples directory.

For Example:

C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\Interop\Basic\ASPXToADO

 

Building the Sample

In order to properly build this sample, the following steps need to be performed prior to running the sample:

  1. Set the ADOLOCATION environment variable to the location containing msado15.dll. No quotes are necessary when setting the ADOLOCATION environment variable.   
  2. If SQL Server is not installed on the local machine or there is an SQL password open InteropADOCS.cs source file and make the changes as described on Line 18 and 27. Open ADOVB.aspx and make changes to Line 27 as described in source file. Save the changes.
  1. Type nmake all from the command line.

For Example:

    SET ADOLOCATION=C:\Program Files\Common Files\System\ado

     NMAKE ALL

 

Running the Sample

In order to run this sample, follow the instructions listed below.

·         Use Internet Explorer to browse the ASP files in the ASPXToADO directory.

For Example:

http://localhost/NetSDK/ASPXToADO/ADOCS.aspx

 or

http://localhost/NetSDK/ASPXToADO/ADOVB.aspx
