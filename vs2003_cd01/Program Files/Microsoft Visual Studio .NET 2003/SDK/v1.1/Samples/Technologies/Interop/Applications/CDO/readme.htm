CDO Interoperability Sample

This document contains the release information for the CDO Interoperability sample.  You will find the following sections below:

 

  • Sample Overview
  • Requirements
  • Location of the sample
  • Building the sample
  • Running the sample

 

Sample Overview

This sample demonstrates the use of an existing unmanaged COM component (Collaborative Data Objects 1.2.1) from managed code.  This sample contains two graphical projects. 

Requirements

This sample requires Microsoft Windows 2000 or above, Collaborative Data Objects (CDO) 1.2.1 and a Microsoft Exchange profile to function properly.  CDO is installed with Microsoft Outlook 2000 by selecting the Custom Installation feature.  Select CDO to run from your system.

Location of the Sample      

This sample is located in the Interop\Applications\CDO directory of the .NET Framework SDK samples directory.

For Example:

C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\Interop\Applications\CDO  

This sample is contained in two subdirectories:

InboxViewer – Using CDO from a C# form

InboxViewer2 – Using CDO from a VB6 form, passing data back and forth between a C# server
 

Building the Sample

In order to properly build this sample:

  1. At the command line enter the following:
    • NMAKE All

 

Running the Sample

In order to run this sample, follow the instructions listed below.

Step 1:

·        Change to appropriate InboxViewer directory.

·        Type InboxViewer from the command line.

For Example:

cd InboxViewer2

InboxViewer 

Step 2:

·        The InboxViewer samples populate a ListView control with messages from the user’s inbox.  To view a message, double-click on an item in the list.

·        Note: A “Choose Profile” dialog may appear at the start of the applications.  Select OK and continue.
