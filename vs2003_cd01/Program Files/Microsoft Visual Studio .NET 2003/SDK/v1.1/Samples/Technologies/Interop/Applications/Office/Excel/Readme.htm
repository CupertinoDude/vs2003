Microsoft Excel Interoperability Sample

This document contains the release information for the Excel Interoperability sample.  You will find the following sections below:

 

 

  • Sample Overview
  • Requirements
  • Location of the Sample
  • Building the Sample
  • Running the Sample

 

Sample Overview

This sample demonstrates using Microsoft Excel from a managed application. Microsoft Excel is the spreadsheet component of Microsoft Office. Microsoft Excel programmatic functionality is exposed through Automation via its type library. The purpose of this sample is to demonstrate that a managed application can interact with Excel as a COM server.

Requirements

This sample requires Microsoft Office 2000 or later to be installed on your system.

Location of the Sample

This sample is located in the Interop\Applications\Office\Excel directory of the .NET Framework SDK samples directory.

For Example:

C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\Interop\Applications\Office\Excel

 

Building the Sample

In order to properly build and run this sample, the environment variable, ExcelTlb, must be set up to point to the location of Excel’s type library:

 

·         If you are using Office 2000, set the variable to the location of excel9.olb.  If you are using OfficeXP, then use the location of excel.exe.

For Example:

set ExcelTlb=C:\Program Files\Microsoft Office\Office\excel9.olb

or

set ExcelTlb=C:\Program Files\Microsoft Office\Office10\excel.exe

·         Once the environment is properly established then type NMAKE from the command line.

For Example:

nmake

 

Running the Sample

In order to run this sample, follow the instructions listed below.

  • Type AutoExcel from the command line.

For Example:

AutoExcel

 

·         At the end of execution the sample leaves the Excel window open to show the results.  Switch to the console window where the sample runs and press ENTER to close Excel and finish the sample.
