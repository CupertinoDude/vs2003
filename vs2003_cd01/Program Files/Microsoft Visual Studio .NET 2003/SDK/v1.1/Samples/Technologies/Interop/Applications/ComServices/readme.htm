Sample Overview

This sample demonstrates the integration between the Microsoft .NET Platform and .NET Framework Services.

COM+ Services is a Microsoft technology that allows the development of highly scalable, multi-tier distributed applications.  Successful commercial Web sites are experiencing ten-fold and even one hundred-fold increases in order volume as more and more business moves to the Internet. It is clear that scalable software systems that remain ready to serve at any time and never fail are a requirement.

This sample demonstrates that your existing technology investment is protected whilst allowing you to move your software forward to take advantage of the innovations in the Microsoft .NET Platform.

 

The Bank Sample demonstrates a simple business problem of performing database operations and preserving the integrity of the database through distributed transactions.

The Bank App Client uses the MoveMoney component to transfer money between two accounts and uses two Account components to perform the transfer. The SQL Server database holds the account rows and also keeps track of receipts. The Account components perform the actual work of updating the database table. MoveMoney and Account are marked to require a transaction. When MoveMoney is created a transaction is started by COM+ Services. When the Account components are created, this transaction is flowed to them, so that when the transfer is completed and MoveMoney returns, the transaction is committed and the database table reflects the transfer.

The components are provided in multiple languages:

·         Bank App client is written in VB6

·         MoveMoney is provided in multiple languages: VB6, VB7, C# and Managed Extensions for VC7

·         Account is also provided in multiple languages: VB6, VB7, C# and Managed Extensions for VC7

The components written in VB7, C# and Managed Extensions for VC7 demonstrate that an existing component can be replaced without amending the client application at all, by performing the same tasks using the same interfaces.

 

Requirements

This sample requires the installation of the following software to function properly:

·         Windows 2000 Professional or Server

·         Microsoft ActiveX Data Objects version 2.5 or later

·         Visual Basic 6.0 with Service Pack 3.0 or later

·         SQL Server 7.0 or later

 

Location of the Sample

This sample is located in the Interop\Applications\ComServices directory of the .NET Framework SDK samples directory.

For Example:

C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\Interop\Applications\ComServices

 

Preparing the Environment

The environment needs to be prepared in order for this demo sample to run.

Database:

You must create a SQL server database called “Bank”. Use the SQL Server Enterprise Manager to create the database. Here are detailed steps:

  • Select    Start -> Programs -> Microsoft SQL Server 7.0 -> Enterprise Manager
  • Open Microsoft SQL Servers
  • Open SQL Server Group
  • Open your Server
  • Right Click on Databases and click “New Database . . .”
  • Enter “Bank” for the database name
  • Click OK

The SQL Server Database “Bank” should have been created.

File DSN:

You must create a File DSN “BankSample.dsn” which points to the “Bank” SQL Server Database. The File DSN “BankSample.dsn” is used by the samples components to update account balances and keep track of receipt numbers. Here are detailed steps:

  • Select    Start -> Settings -> Control Panel -> Administrative Tools -> Data Sources (ODBC)
  • Click on the “File DSN” tab
  • Click the Add button
  • Select SQL Server from the list of drivers and Click Next.
  • Type in the name of the Data Source “BankSample” and Click Next.

·         Click Finish

  • Enter “BankSample” in the Description EditBox.
  • Enter the name of the machine that has the “Bank” database or select from the drop down list. Click Next.
  • Leave authentication to the default “With Windows NT authentication using the network login ID”. Click Next.
  • Change the default database to “Bank”. Click Next.
  • Click Finish.
  • Click Test Data Source
  • Click OK
  • Click OK
  • Click OK

The File DSN “BankSample.dsn” should have been created

 

Building the VB6 MoveMoney and Account Components

First build the VB6 MoveMoney and Account components. Here are detailed steps:

  • Using Windows Explorer locate the Visual Basic Project VB6Bank and double click it.
  • You should now be in the Visual Basic 6.0 development environment. Select File -> Make vbacct.dll.
  • Select Ok to make the dll.
  • You may need to select Yes to overwrite the existing vbacct.dll.
  • Exit Visual Basic.
  • You may need to select OK to save changes to the vb6bank.vbp.

The VB6 MoveMoney and Account components should have been created in the vbacct.dll.

 

Building the Sample

In order to properly build and run this sample, the following steps need to be performed prior to building and running the sample:

  1. Edit Makefile and store the ADO directory in the _ADO_DIR macro.

·         cd C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\Interop\Applications\Comservices

·         notepad Makefile

·         _ADO_DIR=$(ProgramFiles)\Common Files\System\ado

2.      At the command line enter the following:

·         NMAKE All

 

Running the Sample

In order to run this sample, follow the instructions listed below.

1.     At the command line enter the following:

·         cd C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\Interop\Applications\Comservices

·         nmake run

2.      The sample will be loaded in the Visual Basic 6.0 development environment during the last steps of the build process.

3.      Press F5 to execute the unmanaged Visual Basic 6.0 Bank Client for the sample.

4.      Do the following:

       Credit Transaction – Account

In this Scenario we credit account number 1 with $1.

·         Set the Account number to 1

·         Set Amount to 1

·         Set Transaction Type to “Credit”

·         Set Component to Account

·         Set Language to VB6

·         Click Submit

The result box should state that a credit has been applied to account 1 and should display the new balance.

 

Try the Credit operation in VB7

·         Set Language to VB7

·         Click Submit

 

Try the Credit operation in C#

·         Set Language to C#

·         Click Submit

 

Try the Credit operation in VC7

·         Set Language to VC7

·         Click Submit

 

Transfer Transaction – Money

In this Scenario we transfer $5 from account number 1 to account number 2.

·         Set Component to MoveMoney

·         Set Transaction type to “Transfer”

·         Set the Account number to 1

·         Set the Transfer to Account to 2

·         Set Amount to 5

·         Set Language to VB6

·         Click Submit

The result box should state that a credit has been applied to account 2 and should display the new balance for account 2. It should also show that a debit has been applied to account 1 and should display the new balance for account 1.

 

Try the Transfer operation in VB7

·         Set Language to VB7

·         Click Submit

 

Try the Transfer operation in C#

·         Set Language to C#

·         Click Submit

 

Try the Transfer operation in VC7

·         Set Language to VC7

·         Click Submit
