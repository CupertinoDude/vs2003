This document contains the release information for the .NET Framework MSIL sample. You will find the following sections below:

  • Overview
  • Location
  • Building the sample
  • Running the sample

Overview

This sample demonstrates the use of ILDasm and ILAsm to "round-trip" an existing assembly. It is possible to run ILDasm on an existing managed executable to create compileable (and modifiable) MSIL. Then, ILAsm can be used to compile the MSIL source code.

Location

This sample is located in the Technologies\IL subdirectory of the .NET Framework SDK samples directory.

For Example:

C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\IL

Building the Sample

In order to properly build and run this sample, the following steps need to be performed.

Type BUILD.bat from the command line.

For Example:

C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\IL>Build.bat

Running the Sample

In order to run this sample, follow the instructions listed below.

  • Type Roundtrip.bat [FileName] from the command line, where [FileName] is the name of a managed executable.

    For Example:

    C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\IL>RoundTrip.bat Simple.exe

    For Example:

    C:\Program Files\Microsoft.NET\SDK\v1.1\Samples\Technologies\IL>RoundTrip.bat ..\Threading\JoinThread\CS\JoinThread.exe

    Note: It would be necessary to build the JoinThread sample first before attempting to execute the previous usage example.

  • Roundtrip.bat produces two files of interest: RTAsm.il which contains the generated MSIL source code, and RTExe.exe which is a new managed exe built from the RTAsm.il sources.

Notes

For the .Net Framework samples to function properly, your Path, Include, and Lib environment variables must be set correctly. In many cases, the SDK install program updates these environment variables for you. However, if you installed the .Net Framework SDK using Visual Studio .NET, or unselected "register environment variables" during setup, these environment variables will not be updated. You can set the Path, Include and Lib environment variables by running SDKVars.bat, located in the <SDKRoot>\Bin directory. SDKVars.bat must be executed in every command shell.
