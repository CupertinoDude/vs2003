Sample Overview

Desperately waiting for the release date of the NDP, it seemed appropriate to write an application that counts down the second left. It is called Microsoft .NET CountDown. The following list shows the features of the application:

 

  • Displays the seconds left in a window (not the console) with appropriate labeling and updates the display every second until zero is reached.
  • Has a start/stop button that allows the count down to halt or resume.
  • Well, let’s be honest ship dates do move. So, we need functionality to enter a new value for the seconds left using the GUI.
  • Rejects values entered for the number of seconds that are not positive and valid integers.
  • Has resizing behavior for all elements.
  • Produces a beep every second, if present using the speakers, and a final sound when the count reaches zero.

 

Even though not very impressive for a C# programmer, this feature list can be a challenge to write in assembly. However the features of the NDP make it a piece of cake even in assembly language.

 

The program is split into components and modules. This way the various pieces become usable, independent of each other. The window itself, the text box, the button, and the two labels form the UI components. In addition, there are components in the backend that represent the application, the counter, the count value, the exceptions, the error values, and the various events happening in the application. The application forms one assembly that consists of three modules written in the MSIL assembly language:

 

  • CountDown.exe
  • CountDownComponents.dll
  • Counter.dll

 

The component displaying the text “Seconds left” is written in C# and comes from the module CountDownSecondsLabel.netmodule. The component displaying the error messages at the top of the window, one of which is shown in the screenshot above, is available both in C++ and C#. The module is called CountDownErrorLabel.netmodule. Using the C++ version will create not verifiable code. The beeps are produced using native code from the Windows library user32.dll (if Windows9x is used, replace the reference to the DLL by user.dll).

 

Location of the Sample

This sample is located in the CountDown directory of the SDK tool developers guide sample directories.

For Example:

C:\Program Files\Microsoft.NET\FrameworkSDK\Tool Developers Guide\Samples\CountDown\

 

Building the Sample

To build this sample, the following steps need to be performed…

·         To build using C# run compile.bat from the command line.

For Example:

C:\Program Files\Microsoft.NET\FrameworkSDK\Tool Developers Guide\Samples\CountDown\:>compile.bat

·         To build using C++ run “compile.bat C++” from the command line.

For Example:

C:\Program Files\Microsoft.NET\FrameworkSDK\Tool Developers Guide\Samples\CountDown\:>compile.bat C++

 

Running the Sample

In order to run this sample, follow the instructions listed below.

  • Type CountDown.exe from the command line.

For Example:

C:\Program Files\Microsoft.NET\FrameworkSDK\Tool Developers Guide\Samples\CountDown\:>CountDown.exe
