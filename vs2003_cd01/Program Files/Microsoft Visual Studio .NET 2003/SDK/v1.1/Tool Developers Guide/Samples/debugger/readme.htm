CLR Command-Line Debugger (cordbg) Sample

 

Sample Overview

This sample provides the source files for most of the CLR command line debugger, cordbg. These sources demonstrate how to use the CLR Debug interfaces. As a corollary, they also show usage of the CLR MetaData, Symbol Store, and Process Publishing interfaces.

 

Building the Sample

To build the cordbg sample you will need to have a fairly recent version of the Win32 SDK installed on your system. The sample was intended to be built (via nmake) from the version of the C++ compiler that comes with the .NET SDK. Moreover, the sample is dependent on various CLR headers/libs. Below is the list of files included for the sample.

 

\debugger

   makefile

   readme.htm

   commands.cpp

   cordbg.cpp

   debuggerutil.cpp

   debuggerutil.h

   dshell.cpp

   dshell.h

   shell.cpp

   shell.h

   source.cpp

   stdafx.cpp

   stdafx.h

   __file__.h

   cordbpriv.h

   corver.h

   utf.h

   utf.c

 

 

·         Ensure that the environment (command shell) is setup to build using nmake. The makefile assumes that the location of the CLR and Win32 headers/libraries are included in the Win32 environment variables INCLUDE and LIB. The .NET SDK setup will modify these environment variables to include the location for the CLR headers/libraries.

 

·         By default, the makefile assumes that the Win32 headers/libs can be found from the environment variables

 

LIB

INCLUDE

 

If necessary, modify these two environment variables to reflect where the Win32 headers/libs are located on your system.

 

·         By default, the makefile also assumes that the CLR headers/libs can be found from the environment variables

 

LIB

INCLUDE

 

Again, if necessary, modify these two environment variables to reflect where the CLR headers/libs are located on your system.

 

·         From the directory containing the makefile, invoke

 

nmake

 

to build the sample. The end result should be a debug version of cordbg.exe. The makefile can be modified to build a retail version if desired.

 

 

Running the Sample

Please see the Runtime Debugger (cordbg) section in the .NET Framework SDK help file for information on starting and using cordbg.

 

Note. Disassembly is not supported for this sample.
