Configuring Internet Protocol version 6.0

Internet Protocol version 6 (IPv6) is the latest update to the Internet protocol that is used by the Windows operating system, .NET Framework 1.1, and Visual Studio .NET 2003 to enable the next generation of internet applications.

Prerequisites for using IPv6

    You must have Windows Server 2003 or later or Windows XP SP1 or later in order to use IPv6. The latest updates to Windows XP are available at:     http://www.microsoft.com/windowsxp/pro/downloads/default.asp.

Enabling IPv6 in Windows

    By default, IPv6 is not installed and enabled in Windows XP SP1 or later or in Windows Server 2003 or later. For instructions on how to configure IPv6, see the topic      "Checklist: Installing and Configuring IPv6" in Windows Help.

Enabling IPv6 in .NET applications

    Visual Studio .NET and the .NET Framework support network application development on a machine with IPv6 installed. It is recommended that developers enable IPv6 on an  application by application basis, which will ensure that other .NET applications do not change behavior.

 To enable IPv6 for a single application

To enable IPv6 for a particular client or Web application, you must make the following changes to your Application configuration file (app.config) or your Web configuration file (web.config):

<configuration>
    <system.net>
        <settings>
            <!-- The following entry enables IPv6 support in the System.Net classes. IPv6 support is predicated on availability of an IPv6 WinSock provider. -->
            <ipv6 enabled="true" />
        </settings>
    </system.net>
</configuration>

 To enable IPv6 for all .NET applications on a machine

IPv6 can also be enabled for the entire machine by editing machine.config in the .NET Framework 1.1 directory n:\winnt\Microsoft.NET\Framework\<version>\config. To do so, uncomment the ipv6 setting element shown below and change the corresponding enabled value from false to true:

Caution: This change affects the behavior of every .NET application installed on this machine. You should only make this change after proper testing.

<configuration>
    <system.net>
        <settings>
            <!-- The following entry enables IPv6 support in the System.Net classes. IPv6 support is predicated on availability of an IPv6 WinSock provider, use of a supporting operating system, and the switch below being set to "true". -->
            <!-- <ipv6 enabled="false"/> -->
        </settings>
    </system.net>
</configuration>

For more information about IPv6 support in the .NET Framework, see "Internet Protocol Version 6" and "IPv6 Addressing" in the .NET Framework portion of the MSDN Library.
